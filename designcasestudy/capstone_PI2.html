Problem Statement

Problem: Polaris Intelligence’s current application already has many functionalities but it lacks a simple and straightforward way for companies to be able to narrow down the scope of their analytics by selecting multiple variables.

Our Solution: Our module will allow Polaris’ customers to select and view data for two variables in a cross table for an aggregated selection of postal codes. This module will efficiently query and display searches in a few seconds while also storing previous queries for easy retrieval and recurrent review. Our scalable solution does not include changing Polaris' existing services, softwares, and data.

Frontend Requirements:

  1. Drag and Drop functionality.
  2. Selection of two variables.
  3. Upload data in CSV form.
  4. View resulting crosstab.
  5. View previous crosstabs.
  6. Integratable into Polaris’ existing tech stacks.

Backend Requirements

  1. Efficiently query based on user input.
  2. Query for aggregated set of postal codes.
  3. Retrieve correct data from PostgreSQL.
  4. Calculate when necessary, e.g. convert %→count.
  5. Construct correct crosstab.
  6. Store past queries.
